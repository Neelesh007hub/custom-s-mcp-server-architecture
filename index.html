Behind the Scenes: How It Works

See the detailed technical process of how a simple user request flows through our secure system to deliver instant results.

üìÅ List Available Files Flow

üîç Flow Summary:

  1. User Request: "Show me available files"
  2. Client Processing: Cursor AI sends secure HTTP request with API key
  3. Server Authentication: MCP Server verifies API key and processes request
  4. S3 Query: Server queries AWS S3 for file list
  5. Data Processing: Server creates table names and caches metadata
  6. Results Return: Formatted file list sent back to user

‚ö° Query Execution Flow

‚ö° Query Execution Summary:

  1. User Query: "Show me top 5 clients by revenue"
  2. Query Processing: Server converts to SQL and translates file names
  3. Database Execution: DuckDB runs query against in-memory data
  4. Results Formatting: Server formats results as clean table
  5. Display: User sees formatted results instantly
